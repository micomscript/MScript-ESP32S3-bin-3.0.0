差分描画仕様

1.1 差分描画とは
画面全体を毎回描き直すのではなく、変化した行だけを再描画する方式。

メリット
・高速化（Serial 通信量が激減）
・画面のチラつきが減る
・ペースト処理が軽くなる
・スクロールが滑らかになる

1.2 必要な要素
static std::vector<String> prevLines;
・前回描画した行のコピーを保持

1.3 差分描画の流れ
・現在の行内容を取得
・prevLines と比較
・違う行だけ描画
・prevLines を更新

1.4 refresh() への組み込みポイント
for (int i = 0; i < maxVisibleLines; i++) {
Serial.printf("\x1b[%d;1H", editorStartLine + i);
}
・ここに「前回と違う行だけ描画する」条件を追加

基本操作仕様

白い四角：カーソル
Enter：改行
Backspace：カーソル前の文字を削除
Delete：使用不可
矢印キー：カーソル移動
Tab：インデント
Ctrl + S：保存
Ctrl + Q：閉じる
Ctrl + X：保存して閉じる
Ctrl + U：保存されている状態に戻す（Undo）
Ctrl + D：1行削除
Ctrl + K：全削除
Tera Term の「編集 → 貼り付け」使用可（数秒かかる）
